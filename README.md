# creapopop

AI 陪伴工具的创作页。用户在此创建并配置 AI 陪伴角色（基础设定、形象生成、人设润色、角色介绍展示页），产出可被平台其他模块消费的角色配置。

> 本仓库当前聚焦「创作页」单页能力，后续将融入完整平台。架构上预留子路径挂载方式，便于平台集成。

## 状态

🚧 初始化阶段 — 工程脚手架与需求规格进行中。

## 技术栈（规划）

- React 19 + TypeScript + Vite
- Zustand（状态管理）+ React Query（数据请求）
- Tailwind CSS v4（样式）
- Node 原生 `http` 反代（后端代理，承载三类模型 key）

## 复用能力

部分模型调用能力复用自既有创作工具（`newcreation`）：

| 能力 | 用途 | key 承载 |
|---|---|---|
| 生图模型 | 角色头像 / 立绘 | 后端 `.env`，前端不持有 |
| 通用聊天模型 | 人设润色 / 试聊预览 | 后端 `.env`，前端不持有 |
| 代码模型 | 角色介绍展示页 UI 生成 | 后端 `.env`，前端不持有 |

## 快速开始

```bash
pnpm install
cp .env.example .env   # 填入真实凭据，.env 不入库
```

## 开发规范

- 一个需求 = 一个 `feat/*` 分支 + worktree，走 PR，不直接推 `main`
- 模型 key 只存后端，前端源码与产物不出现任何 key
- Arca 接口对齐后端契约，前端为消费方
- UI 覆盖空态 / 加载态 / 错误态三态
